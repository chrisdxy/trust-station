# 正道驿站 - 部署更新指南

## 两种部署方式

| 方式 | 适用场景 | 包大小 | 服务器操作 |
|------|---------|--------|-----------|
| 源码更新 | 修改了页面/组件/API | ~650KB | `npm ci` → `npm run build` → `pm2 restart` |
| 快捷更新 | 仅改前端页面，无新增依赖 | ~650KB | `npm run build` → `pm2 restart` |

---

## 方式一：源码更新（推荐，适用所有场景）

### 本地操作

1. 确保本地构建通过：
```bash
npm run build
```

2. 打包（自动跳过 node_modules/.next/uploads 等大文件）：
```powershell
# PowerShell 一键打包
Remove-Item deploy-package.zip -ErrorAction SilentlyContinue
Compress-Archive -Path src,public,package.json,package-lock.json,next.config.js,tsconfig.json,ecosystem.config.js,.env -DestinationPath deploy-package.zip -Force
```

3. 上传 `deploy-package.zip` 到服务器（覆盖旧的）

### 服务器操作

```bash
# 1. 进入应用目录
cd ~/zhengdao

# 2. 解压覆盖（-o 强制覆盖）
unzip -o ~/deploy-package.zip

# 3. 安装依赖（如果 package.json 没变则很快）
npm ci

# 4. 重新构建
npm run build

# 5. 重启应用
pm2 restart zhengdao

# 6. 检查状态
pm2 status
pm2 logs zhengdao --lines 20
```

---

## 方式二：快捷更新（仅页面改动，无新增 npm 包）

如果只是修改了 `src/` 下的页面代码，依赖没变，可跳过 `npm ci`：

```bash
cd ~/zhengdao
unzip -o ~/deploy-package.zip
npm run build          # 只重新构建
pm2 restart zhengdao   # 重启生效
```

---

## 方式三：单文件热更新（紧急小修复）

只改了一两个文件，不想完整构建：

```bash
# 1. 直接替换文件（scp 或手动编辑）
cd ~/zhengdao

# 2. 重新构建
npm run build

# 3. 零停机重启
pm2 reload zhengdao
```

> `pm2 reload` 与 `pm2 restart` 的区别：reload 逐个重启进程，零停机。

---

## 常见问题

### 构建失败
```bash
# 查看完整错误
npm run build 2>&1 | tail -50

# 清缓存重试
rm -rf .next
npm run build
```

### 端口被占用
```bash
lsof -i :5000
kill -9 <PID>
pm2 restart zhengdao
```

### PM2 进程异常
```bash
pm2 delete zhengdao
npm run build
pm2 start ecosystem.config.cjs
pm2 save
```

### 确认部署成功
```bash
# 本地测试
curl -s http://localhost:5000 | head -5

# 查看运行状态
pm2 status
pm2 monit
```

## 文件对照

| 本地 | 服务器 | 说明 |
|------|--------|------|
| `deploy-package.zip` | `~/deploy-package.zip` | 上传的部署包 |
| `src/` | `~/zhengdao/src/` | 源码 |
| `public/` | `~/zhengdao/public/` | 静态资源 |
| `.env` | `~/zhengdao/.env` | ⚠️ 覆盖前备份服务器版 |
| `ecosystem.config.js` | `~/zhengdao/ecosystem.config.cjs` | PM2 配置 |

> **注意**：`.env` 打包时包含本地配置。如果服务器数据库/密钥不同，解压后先修改 `.env` 再构建。
