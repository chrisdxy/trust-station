# 正道驿站 - 腾讯云服务器完整部署指南

> 适用场景：服务器重装系统后，从零开始重新部署
> 部署包：`deploy-package.zip`（已上传到服务器）

---

## 一、服务器基本信息

| 项目 | 值 |
|------|-----|
| 应用端口 | `5000` |
| Node.js 版本 | `22.x (LTS)` |
| 进程管理 | `PM2` |
| 反向代理 | `Nginx`（可选） |
| 部署目录 | `/home/你的用户名/zhengdao/` |

---

## 二、第一步：上传部署脚本

将以下文件上传到服务器（可以用 `scp` 或 SFTP 工具）：

| 文件 | 作用 |
|------|------|
| `server-setup.sh` | 安装 Node.js / pnpm / PM2 / Nginx |
| `deploy.sh` | 解压部署包、配置环境变量、启动应用 |
| `deploy-package.zip` | Next.js standalone 构建产物 |
| `nginx-zhengdao.conf` | Nginx 反向代理配置模板 |

**上传命令示例（本地电脑执行）：**
```bash
# 把文件上传到服务器的 home 目录
scp server-setup.sh deploy.sh deploy-package.zip root@你的服务器IP:~/
```

---

## 三、第二步：执行环境安装

SSH 登录服务器后执行：

```bash
# 添加执行权限
chmod +x ~/server-setup.sh ~/deploy.sh

# 执行环境安装（Node.js + pnpm + PM2 + Nginx）
bash ~/server-setup.sh
```

**安装内容说明：**
- `Node.js 22` — 通过 nvm 安装，支持版本切换
- `pnpm` — 包管理器
- `PM2` — 进程守护，开机自启
- `Nginx` — 可选，反向代理（80 端口访问无需加端口号）
- 防火墙 — 开放 80/443/5000 端口

---

## 四、第三步：部署应用

```bash
# 执行部署脚本（交互式配置数据库和微信登录）
bash ~/deploy.sh
```

**脚本会依次做这些事：**
1. ✅ 检查 `deploy-package.zip` 是否存在
2. ✅ 解压到 `/home/用户名/zhengdao/`
3. ✅ 交互式配置 `.env` 环境变量（数据库、微信 AppID/Secret、JWT 密钥）
4. ✅ 验证部署包结构（server.js / .next / public / .env）
5. ✅ 生成 PM2 配置文件 `ecosystem.config.cjs`
6. ✅ 用 PM2 启动应用
7. ✅ 测试本地访问

---

## 五、环境变量说明（.env）

部署脚本会自动生成 `.env` 文件，关键配置说明：

```bash
# ---- 数据库 ----
DB_HOST=sh-cynosdbmysql-grp-b72pddqo.sql.tencentcdb.com
DB_PORT=20683
DB_USER=Administrator
DB_PASSWORD=你的数据库密码
DB_NAME=zhengdao
DATABASE_URL=mysql://用户:密码@主机:端口/数据库名

# ---- 微信登录 ----
WECHAT_APPID=wx132561151d9c6e02
WECHAT_APPSECRET=你的微信AppSecret
NEXT_PUBLIC_WECHAT_APPID=wx132561151d9c6e02

# ---- JWT ----
JWT_SECRET=自动生成的随机密钥

# ---- 应用 ----
NODE_ENV=production
PORT=5000
HOST=0.0.0.0
NEXT_PUBLIC_APP_URL=http://服务器IP:5000
```

> ⚠️ **注意**：修改 `.env` 后需要重启应用：`pm2 restart zhengdao`

---

## 六、第四步：（可选）配置 Nginx 反向代理

如果不想用户访问时带 `:5000` 端口，可以配置 Nginx 转发 80 端口：

```bash
# Ubuntu / Debian
sudo cp nginx-zhengdao.conf /etc/nginx/sites-available/zhengdao
sudo ln -s /etc/nginx/sites-available/zhengdao /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# CentOS
sudo cp nginx-zhengdao.conf /etc/nginx/conf.d/zhengdao.conf
sudo nginx -t
sudo systemctl reload nginx
```

**修改配置文件**：把 `/home/your_user` 替换为实际的部署路径。

## 七、部署后验证

```bash
# 查看 PM2 状态
pm2 status

# 查看实时日志
pm2 logs zhengdao

# 本地测试（服务器上执行）
curl http://localhost:5000

# 外部访问测试（浏览器）
http://你的服务器IP:5000
```

**预期结果：**
- PM2 状态显示 `online`
- `curl` 返回 HTML（包含 `<html`）
- 浏览器访问能看到页面样式（CSS 正常加载）

## 八、常见问题排查

### 问题 1：页面无样式（CSS 404）
```bash
# 检查 static 目录是否存在
ls /home/你的用户名/zhengdao/.next/standalone/.next/static

# 如果不存在，重新执行本地 build，确保 build.cjs 正确复制了 static
# 然后重新打包上传
```

### 问题 2：微信登录回调失败
检查 `.env` 中的 `WECHAT_APPID` 和 `WECHAT_APPSECRET` 是否正确
```bash
cat ~/zhengdao/.env | grep WECHAT
```

### 问题 3：数据库连接失败
```bash
# 测试数据库连通性
mysql -h sh-cynosdbmysql-grp-b72pddqo.sql.tencentcdb.com \
      -P 20683 \
      -u Administrator \
      -p zhengdao
```

### 问题 4：PM2 重启后应用没起来
```bash
pm2 logs zhengdao --lines 50
# 查看错误日志，一般是 .env 配置问题
```

## 九、快速命令参考

```bash
# 启动
pm2 start ~/zhengdao/ecosystem.config.cjs

# 重启
pm2 restart zhengdao

# 停止
pm2 stop zhengdao

# 查看日志（实时）
pm2 logs zhengdao --lines 100

# 查看状态
pm2 status

# 开机自启（执行一次即可）
pm2 startup
pm2 save
```
